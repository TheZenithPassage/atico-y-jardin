# Friday Demo Script

Use this script for the Friday, 2026-07-03 owner demo.

## Main Message

Ático y Jardín will start as a polished public website for the two apartments,
then grow into a private reservation-management tool using the inherited
technical base.

## What To Show First

Start with the public landing. Do not open inherited private CatWorld screens
first unless asked about the future admin roadmap.

## Demo Flow

1. Home.
2. Ático section.
3. Ático gallery.
4. Jardín section.
5. Jardín gallery.
6. Location/contact.
7. Future private management roadmap.

## Suggested Phrases

- "The first release focuses on presenting the two apartments clearly and professionally."
- "Ático and Jardín are separated inside one shared brand, so each apartment can have its own photos, details, and contact CTA."
- "For Friday, the inquiry area is intentionally simple. It is a contact or availability request path, not an automatic booking engine."
- "The private reservation system is planned as the next phase, using the existing technical foundation rather than starting from zero."
- "A public request will not block dates automatically. It will be reviewed before it becomes a confirmed reservation."

## What Not To Promise

- Do not promise online payment for the Friday demo.
- Do not promise automatic availability blocking.
- Do not promise Booking.com, Airbnb, iCal, or channel-manager synchronization.
- Do not promise that all content, photos, SEO, legal copy, or multilingual copy is final.
- Do not present inherited CatWorld private screens as finished Ático y Jardín reservation management.

## If They Ask About Reservations

Recommended answer:

"The reservation-management area is the next release. The inherited system
already has useful pieces such as login, calendar, date validation, and overlap
checks. We will adapt those carefully to apartments after the public demo, so
we do not mix a visual launch with risky domain refactoring."

## If They Ask About Payments

Recommended answer:

"Payments are intentionally not part of the first releases. First we need the
public presentation, then reliable private reservation management, then public
requests. Payment can be assessed later if it is genuinely needed."

## If They Ask About Airbnb Or Booking.com

Recommended answer:

"External channel synchronization is not part of the first demo or the first
reservation alpha. We can document it as a later integration candidate, but it
should not delay the public website or the private reservation workflow."

## Risks During Demo

- Real photos may still be placeholders.
- Some content may still show TODOs until the owners provide exact details.
- Private reservation features may still be inherited/planned rather than
  fully adapted.
- Legal/privacy copy may be placeholder until production preparation.

## Closing

End by showing the release path:

- v0.1.0: public landing for the owners to review.
- v0.2.0: private reservations.
- v0.3.0: public booking requests reviewed internally.
- v0.4.0: owner review beta with real content and staging.
- v1.0.0: production launch.
